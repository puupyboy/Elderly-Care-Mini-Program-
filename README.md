# Elderly-Care-Mini-Program-
  本小程序是完全由个人开发的，是我的第一个小程序也是我的第一个稍微大一点的项目，存在许多不足。但好处是能用，但是好处是我的readme记录了比较完整的开发过程，并且刚刚开发完还记得几乎所有遇到的问题，适合懂一点代码但是没开发过小程序的人看我的readme学习一下完整流程 :)  
  这是一款养老平台微信小程序，包括吃药提醒，每日健康记录，养生视频观看，商城功能，AI功能与VIP功能与登录功能。目前小程序已经正式上架，本人负责全部的开发任务，readme记录了具体开发过程和出现的错误与解决方法。这是本人制作的第一款小程序  
  具体下载使用需要把agent-ui文件夹和toolCard文件夹合并新建一个文件夹命名为components，project.private.config文件涉及私用id没有上传，需要使用的可以自行创建搜索代码如何编写、编写好放在与app.js等并列即可。  
  此小程序采用微信云开发，可以先注册一个APPID，然后导入本小程序，即可查看并修改，其中志愿者模块可以自行在云存储那里添加或减少，需要播放的视频也都可以自行添加到云存储，因可能涉及版权问题所以我没放进去。这一版本吃药提醒没有开发，后续可能进行修改。  
  笔者创建小程序心路历程如下（创建的具体过程见下一段）：本人吉林大学大二非计算机相关专业，但考虑可能日后转码因此找到了一家初创小公司实习，这家公司主要业务是养老相关，盈利点是小程序的VIP功能和与另外一家食品公司合作销售其绿色视频，完全的ToC公司。小程序的开发首先公司要求开发网页并非小程序，并告诉我后端有专门的人负责，希望让我负责前端，于是我用React语言写了网页前端，同时打算与后端一起讨论接口相关问题。但后端据说没几天就跑路了，且公司要求创建小程序不创建网页了要创建小程序，且前后端都由我自己一个人负责，感觉这样安排不是很合理，但想着增长经验还是继续干了。把小程序建好后，剩下的商城盈利等功能由另外一个人负责，我就没有参与了，因此本人分享的经验比较适合个人建立一个完整的非盈利性的小程序。  
  具体创建过程：因先使用React写网页的前端基本上已经实现差不多了，但让我改成小程序不要网页了，我搜索后发现了Taro可以把网页转换成小程序，并且要采用微信云开发功能而不是直接使用测试号进行微信小程序开发可以规避一些我不熟悉的后端（btw云开发规避了后端，前端还能由ai实现大部分，其实不写代码也能基本完成小程序开发了，只是ai写的代码需要能看懂）因此此时我的思路是：用Taro转化成小程序，再学习云开发，看看如何调用后端接口，理论上就能把小程序做出来了。但是实际操作中出现了如下几个难点：1.Taro转换并不是那么完美，仍然需要手动修改。2.接入ai采用的是dy平台的扣子ai，不能接入tx小程序，因此需要重新看看如何实现ai功能。3.网页版React开发用的UI小图标不能继续用在小程序里面了，React能直接调用小图标但是小程序必须转化成图片的形式了，多了一点麻烦。4.云开发如何调用后端的数据等需要学习。5.云开发图片的大小不能超过1M，而且所有的美工都需要我自己去做，因此需要先把小程序的雏形做好后加入贴图把小程序美化。6.如何与另外一个负责商城收付款和制作数据集的人合作，我需不需要用特定的接口等问题。7.老板告诉我设计的小程序结果也就是个笼统的样子，如何把他的表达变成符合他心情的小程序。8.公司一共十几个人，跟谁汇报我的工作，如何让他们看到自己既是每天都在努力开发，又是具体事情要老板定夺，又显得自己没那么蠢（
